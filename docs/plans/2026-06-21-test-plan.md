## 1. 测试目标

**总目标**：验证重构后的 dashboard 在以下三个核心解耦场景下行为正确：
- 列表增删仅由进程轮询驱动
- 指示灯颜色仅由 CC hook 驱动
- JSONL 仅用于 UI 元数据展示（标题/context%/subtitle），不参与状态判断

**子目标**：
| # | 目标 | 失败模式 |
|---|------|---------|
| O1 | 1 个 alive CC = 1 行；进程消失立刻移除 | 多个 session、闲置消失、不消失 |
| O2 | 7 条 hook 规则全部正确生效 | 颜色错、PERMISSION 卡死 |
| O3 | JSONL 删除/损坏不阻塞列表与状态 | UI 崩溃、状态错乱 |
| O4 | 多 CC 实例同 cwd 各自独立 | 状态串号 |

## 4. 测试设计技术

| 技术 | 用在哪 |
|------|--------|
| **等价类划分** | CC 进程类型（真 CC / claude-mem observer / 其他 exe） |
| **状态转换测试** | 7 条 hook 规则覆盖所有状态边 |
| **决策表** | hook event × 当前 status = 新 status |
| **边界值** | PID 范围、port 范围、JSONL 文件大小 |
| **错误猜测** | 端口被占、cc 进程闪退、hook curl 失败 |

## 11. 风险登记

| 风险 | 影响 | 概率 | 缓解 |
|------|------|------|------|
| 测试套件被死 import 卡住 | L1 不能跑 | 100%（已发生） | 用户批准前用 try/except 或 skip 标记解决 |
| HookServer 测试端口冲突 | L2 不能跑 | 中 | 用端口 0 让 OS 自动分配空闲端口 |
| pytest-qt 在 headless 环境失败 | L3 不能跑 | 中 | 用 `pytest-qt`'s `qtbot` fixture 或标记 `pytest.mark.skipif` |
| claude-mem observer 路径变化 | TC-009 失败 | 低 | 测试用固定的 cmdline 字符串匹配，不依赖具体路径 |
| `$CLAUDE_SESSION_ID` 不存在 | TC-E2E-001 失败 | 中 | 文档明确说明，测试用直接 curl 发 sid |
